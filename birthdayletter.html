Here is a ROT13 function in Python I found from https://www.dotnetperls.com/rot13-python. 

def rot13(s):
    result = ""

    # Loop over characters.
    for v in s:
        # Convert to number with ord.
        c = ord(v)

        # Shift number back or forward.
        if c >= ord('a') and c <= ord('z'):
            if c > ord('m'):
                c -= 13
            else:
                c += 13
        elif c >= ord('A') and c <= ord('Z'):
            if c > ord('M'):
                c -= 13
            else:
                c += 13

        # Append to result.
        result += chr(c)

    # Return transformation.
    return result

Cipher’s are a subset of encryption where each character in the source text is replaced by another character. A property all ciphers share is that they don’t change the shape of the message, only the characters. It’s easy to see this property when comparing the original to the ROT13 result: the punctuation, spacing, and the number are exactly the same.

ROT13 is an example of one such cipher. Specifically, ROT13 is a symmetric substitution cipher. I don’t know any other symmetric substitution ciphers off the top of my head, so I decided to broaden my thoughts to any substitution cipher, symmetric or not. All substitution ciphers share a property from which they get their name: Each character maps to exactly one other character, so if the letter “a” in the source is replaced by a “b” in one place, every instance of “a” will be replaced by “b”.

With a Vigenère cipher, much like a Caesar cipher, the letter to replace is found by shifting the alphabet by a certain amount. def vigenere_offset(source_index, key):
    return ord(key[source_index % len(key)]) - ord('a')

def vigenere_encode_char(source_index, source_char, key, salt = 0):
    offset = vigenere_offset(source_index, key)
    ord_source = ord(source_char) - ord('a')
    shifted = (ord_source + offset + salt) % 26
    return chr(shifted + ord('a'))

print("'%s' should be 'l'" %vigenere_encode_char(50, "a", "lemon"))

'l' should be 'l'

Decoding should be the opposite of encoding: 

def vigenere_decode_char(encoded_index, encoded_char, key, salt=0):
    offset = vigenere_offset(encoded_index, key)
    ord_encoded = ord(encoded_char) - ord('a')
    shifted = (ord_encoded - offset - salt) % 26
    return chr(shifted + ord('a'))

print("'%s' should be 'a'" %vigenere_decode_char(50, "l", "lemon"))

'a' should be 'a'

Now we need a function to run through the whole source message and encode it. I’ll explain the reason for the optional index and salt parameters later: 

def vigenere_encode_message(source, key, encoding_index = 0, salt = 0):
    encoded = ""
    for char in source:
        # If the character is alphabetic
        if char >= 'a' and char <= 'z':
            # Encode before appending
            encoded += vigenere_encode_char(encoding_index, char, key, salt)
            # Then shift the vigenere cipher
            encoding_index += 1
        else:
            # Just append
            encoded += char
    return encoded
